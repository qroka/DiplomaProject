# Кадровый портал Сургутского района — UX-исследование и рекомендации по дизайну

**Дата:** 30 июня 2026  
**Метод:** Экспертная оценка (эвристический аудит) кодовой базы, анализ информационной архитектуры, сравнение с практиками госсектора и WCAG 2.1 AA  
**Объект:** Frontend (Nuxt 3 + Nuxt UI), разделы навигации по ТЗ

> **Ограничение методологии:** отчёт основан на анализе реализации и типичных паттернов поведения целевой аудитории. Для подтверждения гипотез рекомендуется провести 8–12 модерируемых юзабилити-сессий и замерить воронку «просмотр вакансии → отклик».

---

## Краткое резюме

Портал имеет **сильную функциональную основу**: полная навигация по ТЗ, форма отклика с юридическими согласиями, версия для слабовидящих (ГОСТ Р 52872-2024), skip-link, тёмная тема. Основные риски для пользователя — **перегруженная навигация**, **незавершённый контент** (заглушки на контактах и доске почёта), **слабая визуальная иерархия CTA** на ключевом пути «найти вакансию → откликнуться», **однообразие визуала** (одно hero-изображение на всех страницах) и **мобильные огрехи** (маркиз логотипов, плотное меню из 9 пунктов).

**Топ-3 приоритета:**
1. Завершить контент и убрать заглушки — доверие к госсайту критично.
2. Упростить путь к отклику и усилить визуальный приоритет «Вакансии» / «Откликнуться».
3. Унифицировать дизайн-систему: типографика, отступы секций, состояния карточек и фильтров.

|----------|----------|

## Карта пользовательского пути (Jobs-to-be-Done)

### Путь A: Поиск и отклик на вакансию

```
Главная → [Hero CTA «Наши вакансии»] → Список вакансий → Фильтры → Карточка → Подробнее / Отклик → Модальная форма → Успех
```

| Этап | Наблюдение | Эмоция | Возможность |
|------|------------|--------|-------------|
| Главная | Hero с понятным CTA — хорошо | Уверенность | Добавить счётчик «N открытых вакансий» для срочности |
| Список | 3 dropdown-фильтра без счётчика результатов | Неопределённость | Показать «Найдено: N», чипы активных фильтров |
| Карточка | Две кнопки одинаковой ширины; «Откликнуться» визуально слабее из-за дублирования стилей | Сомнение | Primary = отклик, secondary = подробнее |
| Форма | 4 длинных чекбокса согласия — необходимо по закону | Усталость | Прогресс-бар, группировка, ссылка «что это значит» |
| Мобильная форма | Модальное окно на весь экран — ок, но длинный скролл | Раздражение | Sticky footer «Отправить» + якоря по секциям |

### Путь B: Ориентация в структуре администрации

```
О нас → Цитата главы → Структура (DeputyStructure) → Доска почёта / Кадровый резерв
```

| Этап | Наблюдение | Проблема |
|------|------------|----------|
| Доска почёта | Hero: `title="Test"`, `description="testtesttest"` | **Критический разрыв доверия** |
| Контакты | Та же заглушка | Пользователь не может дозвониться с уверенностью |
| О нас | Качественный контент и соцсети | Хороший эталон для остальных страниц |

### Путь C: Молодёжь и кадровый резерв

Табы (`UTabs`, pill) — удачный паттерн для длинного контента. Риск: **вторичные CTA** («Перейти к конкурсам») теряются внизу длинных секций — нужны промежуточные якоря.

## Результаты эвристической оценки

### Информационная архитектура

| Критерий | Оценка | Комментарий |
|----------|--------|-------------|
| Полнота разделов ТЗ | ✅ | 9 пунктов в `mainNavItems` |
| Глубина навигации | ⚠️ | Все разделы на одном уровне — когнитивная нагрузка |
| Поиск | ❌ | Глобального поиска по вакансиям/конкурсам/новостям нет |
| Хлебные крошки | ⚠️ | Только на странице вакансии; на вложенных «О нас / департаменты» — проверить единообразие |

**Рекомендация:** сгруппировать меню в 2 уровня (desktop: mega-menu или dropdown «О кадрах» / «Карьера» / «Прозрачность»). На mobile — аккордеон с группами.

### Визуальный дизайн и бренд

| Элемент | Текущее состояние | Рекомендация |
|---------|-------------------|--------------|
| Цвета | Зелёный `#50C891` + синий `#2D2382`, градиент `bg-gradient-brand` | Сохранить; зафиксировать токены в `DESIGN.md` |
| Шрифт | Verdana (системный) | Для госсайта допустимо; рассмотреть **PT Sans** / **Inter** для лучшей читаемости кириллицы |
| Hero-изображение | `/images/office.png` на всех страницах | Уникальные иллюстрации/фото по разделам |
| Разделители | Многократный `h-px bg-gradient-brand` | Вынести в компонент `SectionDivider`; уменьшить частоту |
| Карточки новостей | `bg-secondary-500` + белый текст | Сильный контраст; проверить alt у изображений постов |
| Тёмная тема | Реализована | Проверить контраст primary-500 на тёмном фоне (WCAG) |

### Компоненты и паттерны

#### Шапка (`AppHeader.vue`)
- **Плюс:** skip-link, панель a11y, CTA «Смотреть вакансии» в mobile drawer.
- **Минус:** 9 ссылок в горизонтальном меню на lg — при 1280px возможен overflow/скролл (`overflow-x-auto`).
- **Рекомендация:** выделить «Вакансии» как `UButton` primary в шапке (desktop); сократить видимые пункты до 5–6 + «Ещё».

#### Карточка вакансии (`VacancyCard.vue`)
- Кнопка «Откликнуться» использует `color="green"` но классы `bg-primary-500` — **визуальная путаница**.
- Badge «Новое» без критерия давности в UI (зависит от API).
- **Рекомендация:** единая семантика кнопок; дата публикации на карточке.

#### Фильтры (`DropdownFilters.vue`)
- Нет индикации активного фильтра на кнопке (кроме подписи значения).
- Нет комбинированного сброса всех фильтров.
- **Рекомендация:** чипы под фильтрами + «Сбросить всё».

#### FAQ (`index.vue`)
- Ответы на 2–4 вопроса — **стены текста** (300+ слов).
- Один ответ помечен «не финальный текст».
- **Рекомендация:** краткий ответ в аккордеоне + ссылка «Подробнее»; вычитка юристом/HR.

#### Маркиз партнёров (`index.vue`, TODO в коде)
- На маленьких экранах логотипы «стоят в ряд не шевелясь».
- **Рекомендация:** на `<sm` — статичная сетка 2×2 вместо marquee; `prefers-reduced-motion` уже есть глобально.

### Доступность (a11y)

| Реализовано | Пробел |
|-------------|--------|
| Skip-link, `lang="ru"`, `@nuxt/a11y` | Нет `aria-live` для toast успеха формы |
| Версия для слабовидящих (масштаб, контраст) | Полупрозрачные фоны ломают high-contrast режим |
| `prefers-reduced-motion` | Marquee всё ещё анимируется до срабатывания media query |
| `aria-label` на части кнопок | Фильтры без `aria-expanded` / объявления выбора |

**Рекомендация:** в режиме a11y заменять `bg-white/60` на непрозрачные фоны; добавить `role="status"` для динамических списков вакансий после фильтрации.

### Доверие и контент (критично для госсектора)

Обнаружены **незавершённые страницы**:
- `contacts.vue` — hero «Test» / «testtesttest»
- `honorboard.vue` — то же
- FAQ — нефинальный текст про коллектив

Это **блокер восприятия** — пользователь может решить, что сайт неофициальный или заброшенный.

### Производительность и UX загрузки

- `useAsyncData` с `{ server: false }` на главной и вакансиях → **пустые состояния при первом рендере**, затем подгрузка.
- **Рекомендация:** skeleton для карточек вакансий и новостей; SSR для публичных списков где возможно.

## Предлагаемая дизайн-система (кратко)

### Типографическая шкала

| Токен | Desktop | Mobile | Использование |
|-------|---------|--------|---------------|
| `display` | 48px / 1.1 | 32px | Hero H1 |
| `h2` | 36px | 28px | Заголовки секций |
| `h3` | 24px | 20px | Подсекции, карточки |
| `body` | 16px | 16px | Основной текст (не ниже 16px на mobile) |
| `caption` | 14px | 14px | Мета, даты |

### Вертикальный ритм секций

- Стандартный блок: `py-12 lg:py-16` (сейчас смешаны `py-6`, `py-8`, `mb-6`).
- Максимальная ширина текста: `max-w-3xl` для prose; `max-w-7xl` для сеток.

### Семантика кнопок

| Действие | Вариант | Пример |
|----------|---------|--------|
| Главное действие раздела | `primary` solid | Откликнуться, Наши вакансии |
| Вторичное | `primary` outline или `neutral` outline | Подробнее, Конкурсы |
| Навигация вне контекста | `ghost` / link | Фидбэк, соцсети |

### Карточки

- Единый border: `ring-1 ring-gray-200 dark:ring-gray-800`.
- Hover: `shadow-md` + `translate-y-0.5` (уже частично на VacancyCard).
- В a11y-режиме: отключить backdrop-blur и opacity < 1.
